## 2. Points positifs confirmés (architecture saine)

- **Sécurité / secrets** : aucun secret en clair ; clés Stripe/DB/Resend/Blob en env only ; `getStripe()` lazy (pas de crash à l'import sans ENV).
- **Webhooks** : signature HMAC vérifiée avant tout traitement (400 sinon) ; idempotence par contrainte UNIQUE sur `eventId` (200 silencieux sur doublon) ; rate-limit présent.
- **Statut payant = source de vérité serveur** : `activerReseauPartenaire` vérifie `niveau==='national'` avant d'écrire `partenaire`/`palier` ; jamais posé côté client.
- **Gate de publication hiérarchique** : la publication d'événement passe par `peutPublierEvenement` (= `nationalDe(reseau).partenaire`) — un local hérite de l'abonnement de son national ; refus FR si non abonné.
- **Autorisation délégation** : `/api/admin/deleguer-local` réservé `role==='admin'` (sur user frais), rate-limité ; `skipAutoCreateReseau` empêche l'auto-création d'un national pour un compte délégué.
- **Hiérarchie** : 2 niveaux stricts (parent d'un local doit être national) ; garde suppression national-avec-locaux ; unicité `(user) WHERE niveau='national'`.
- **Affiliation locaux-only** : refus serveur FR si un national est coché dans `reseauxFrequentes`.
- **Cartes** : `/api/geo/reseaux` = locaux uniquement (filtre `niveau='local'` dans les 2 couches), Zod, SQL paramétré, GiST ; un seul type de marqueur événement (Premium retiré).
- **SEO/RGPD** : fiches singulières inchangées (slugs figés) ; canonical des bascules (état `?vue=`/filtres non canonicalisé) ; 301 `/carte/*` ; `parentOrganization` (local→national) sans donnée perso ; JSON-LD échappé XSS ; opt-out + `noindex` par défaut sur réseauteurs non validés ; géoloc ville.
- **ADR-0009** : `groupes` et migrations intactes (dormant).
